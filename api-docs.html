Products

All product endpoints are prefixed with /v1/products.

POST /v1/products

Create a new product. All fields in the request body are validated — unknown fields are rejected.

Request Body

Field Type Required Constraints
name string required Non-empty string
price number required ≥ 0
description string optional —
isActive boolean optional Defaults to true
Request
curl -X POST \
  https://my-app-production-f6bc.up.railway.app/v1/products \
  -H "Content-Type: application/json" \
  -d '{
    "name": "MacBook Pro",
    "price": 1999,
    "description": "Fast laptop",
    "isActive": true
  }'

Product Schema

Field Type Required Default Description
_id ObjectId — auto MongoDB document ID
name string yes — Product name
price number yes — Product price (≥ 0)
description string no — Optional description
isActive boolean no true Active status
createdAt Date — auto Creation timestamp
updatedAt Date — auto Last update timestamp

Validation Rules

Applied globally via NestJS ValidationPipe.

whitelist: true

Strips any extra fields not declared in the DTO silently before processing.

forbidNonWhitelisted: true

Rejects the request with 400 if unknown fields are present.

Common 400 Causes

Mistake Error Message
name sent as a number name must be a string, name should not be empty
price sent as a string price must be a number conforming to the specified constraints
price is negative price must not be less than 0
Unknown field sent (e.g. category) property category should not exist
isActive sent as a string isActive must be a boolean value
